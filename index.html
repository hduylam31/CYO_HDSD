Bước 1 — Cài Node.js

Runtime môi trường · Làm trước tiên

  • 1
    Vào link bên dưới, tải file .msi (Windows) hoặc .pkg (Mac)
    → nodejs.org/en/download  · Chọn mục LTS
  • 2
    Mở file vừa tải, nhấn Next liên tục đến khi xong
    Không cần thay đổi gì — giữ mọi tùy chọn mặc định
  • 3
    Báo IT biết đã cài xong — IT sẽ hỗ trợ các bước tiếp theo
    IT sẽ setup MCP server và kết nối với Claude Desktop cho bạn
⚙️

Bước 2 — Kết nối MCP vào Claude Desktop

Edit config JSON · Sau khi Node.js chạy xong

  • 1
    Mở Claude Desktop → nhấn vào tên mình (góc trên trái) → Settings
    Cần dùng Claude Desktop app (tải tại claude.ai/download), không phải browser
  • 2
    Vào tab Developer → nhấn Edit Config
    File config tự động mở bằng Notepad / TextEdit — đây là file JSON thuần text
  • 3
    Copy đoạn JSON bên dưới, paste vào file (thay thế toàn bộ nội dung cũ nếu có)
    {
      "mcpServers": {
        "cyo-cdp": {
          "command": "node",
          "args": ["/path/to/cyo-mcp-server/index.js"],
          "env": {
            "BQ_PROJECT_ID": "yody-prod"
          }
        }
      }
    }
    ⚠️ Thay /path/to/ bằng đường dẫn thực tế trên máy của bạn đến folder repo vừa clone ở Bước 1
  • 4
    Lưu file → Quit hoàn toàn Claude Desktop rồi mở lại
    Phải quit hoàn toàn (Cmd+Q / Alt+F4), không chỉ đóng cửa sổ — Claude Desktop cần restart để load config mới
  • 5
    Mở lại Claude Desktop → kiểm tra icon 🔌 (MCP connected) xuất hiện trong chat
    Gõ thử: "Chạy health check MCP server" — Claude gọi được tool là thành công ✓
✨

Bước 3 — Cài Skill trong Claude

Custom instructions · Tối ưu output cho CYO context

Skill là gì? Là bộ hướng dẫn giúp Claude hiểu đúng context CYO ngay từ đầu — IT chuẩn bị sẵn file .zip, MKT chỉ cần import vào là dùng được.
  • 1
    Nhận file CYO-skill.zip từ IT (gửi qua Slack / Drive)
    Lưu file về máy, không cần giải nén
  • 2
    Mở Claude Desktop → vào Settings
  • 3
    Chọn mục Connectors → nhấn Customize
  • 4
    Nhấn dấu (+) → chọn Add Skill → chọn file CYO-skill.zip vừa nhận
  • 5
    Skill xuất hiện trong danh sách → nhấn Enable là xong
    Thử ngay: "Top sản phẩm bán chạy tháng 4?" — nếu Claude trả lời được là thành công ✓

KPI Đo Lường & Cách Đo
3 nhóm KPI chính — Qualitative, Performance, Cost
Feedback MKT
≥ 10 feedback
Số lượng phản hồi có mô tả tối thiểu trong W4
Query Latency
< 5s
P95 end-to-end response time
MCP Response
< 5s
Thời gian MCP trả data về Claude (P95)
Token / Query
≤ 3K
Token budget trung bình mỗi query
KPI Loại Công cụ đo Cách thực hiện
Feedback team MKT Quant + Qual Google Form / Notion / Slack thread Đếm số lượng feedback có nội dung mô tả (không tính "OK" / 1 từ). Mỗi feedback cần ghi rõ: query đã hỏi gì, Claude trả lời ra sao, vấn đề gặp phải (nếu có). PO tổng hợp cuối tuần theo 2 cột: Số lượng + Nhóm vấn đề (data sai / chậm / khó hiểu / khác).
Query Latency (E2E) Perf Stopwatch / Chrome DevTools Bấm giờ từ lúc gửi câu hỏi → nhận response đầy đủ. Ghi vào sheet: timestamp, query type, duration (ms).
MCP Response Time Perf Server log / console.time() Log thêm console.time('mcp_query') trong handler. Export log file cuối ngày.
Token tiêu tốn Cost Anthropic Console → Usage Vào console.anthropic.com → Usage tab → filter theo ngày. Ghi lại input tokens + output tokens mỗi ngày testing.
Answer Accuracy Qual Manual spot-check vs BigQuery Chọn ngẫu nhiên 5 queries/ngày, chạy query SQL thủ công trên BQ để verify số liệu AI trả về có đúng không.
Error Rate Quant Server error log Đếm số lần MCP trả error / tổng số queries. Target: < 5% error rate trong W4.
